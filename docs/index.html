Architecture & Design

How ViridisCFA
actually works

An automated equity-research engine that pulls SEC filings, earnings-call transcripts and insider trades, runs a deterministic financial scorecard in Python, and uses GPT-5.4 only for the things language models are actually good at — reading, synthesising and arguing. The numbers are never left to the model.

~4,200 LOC Python 3 parallel data branches 5 deterministic models 4 output formats Served over MCP

01 · The core idea

A deterministic spine, with narrative muscle

The central design decision — the thing everything else hangs off — is a hard line between what a computer must do and what a language model is allowed to do.

Large language models are persuasive but cannot be trusted to do GAAP arithmetic: they round, they "helpfully" recompute, they hallucinate a plausible figure when one is missing. So ViridisCFA computes every financial metric in Python, directly from verified SEC XBRL facts, and then forbids the model from touching those numbers. The model's job is to read the prose a human analyst reads — the filing narrative, the earnings call, the footnotes — and to weigh, connect and argue. Two layers, one barrier between them.

Branches run in a thread pool, so wall-clock time is the slowest branch, not the sum. The filing branch is the heavy one: it generates the scorecard, runs a high-reasoning expert analysis, then a second missing-information pass that acts as an adversarial reviewer of the first. 03 · The deterministic quant engine

Exact numbers — and the judgement to know when they apply

Five model families are computed from XBRL facts with full audit trails. The interesting part is not the math; it is the layer that decides which numbers are meaningful for this company.

The applicability gate — why this exists

Altman assumes working capital and sales-to-assets mean something. Beneish assumes an established firm with stable revenue. Hand those models a bank, a REIT, a pre-revenue biotech or a hyper-growth software company and they return authoritative-looking numbers that are category errors. So before scoring, the engine classifies the issuer — by SIC code, revenue-to-assets and sales growth — and suppresses or caveats the metrics that do not apply.

A separate guard catches Beneish scores that fall outside the model's empirically meaningful band — a tell-tale of a tiny or freshly-listed issuer whose ratios blow up on small denominators. The score is shown, but flagged as a numerical artifact rather than a manipulation signal.

Every value carries an honest status — Programmatically Computed, not "Verified" — plus disclosure of any imputed or derived inputs. The arithmetic is exact; the label never over-claims.

04 · The AI layers

Four passes, each with one job

The model is used in a chain where each step is narrow and checkable, rather than one monolithic "analyse this company" prompt.

Expert analysis

Reads the full filing plus the deterministic scorecard and writes a structured analysis — but is told to flag any metric the scorecard marks not-applicable for the business model, rather than parrot it.

gpt-5.4 · reasoning: high

Missing-information cross-check

A second, adversarial pass: "what did the team miss?" It re-reads the filing against the first analysis and surfaces only what was omitted or wrong — catching gaps a single pass glides over.

gpt-5.4 · reasoning: medium

Final synthesis

Weaves filing, missing-info, transcript and insider data into one memo organised by insight, not by source document. It opens with comparability flags, then argues, then commits — see below.

gpt-5.4 · medium · emits the brief inline

Decision brief

A strict JSON contract — stance, conviction, thesis, upside/risk drivers, catalysts, data-quality flags — emitted in the same response as the memo, so the structured summary can't drift from the prose.

one structured object · falls back to a separate call if needed

Making the stance earn its place

A research tool that always says "Neutral" is useless. The synthesis prompt forces an explicit adjudication step — "The Call" — that weighs which case the evidence actually supports and names the deciding variables, before stating a stance. Neutral is reserved for genuinely two-sided setups and must come with "the single development that would move me off the fence." Conviction is calibrated to how lopsided the evidence is.

In testing, this moved a batch of six story stocks from a reflexive 6×Neutral to a calibrated mix — Bullish on the healthiest name, Bearish (low conviction) on the weakest — with the genuinely balanced names correctly staying Neutral.

05 · Trust, reproducibility & cost

Every report can be audited and cheaply re-derived

Provenance

A footer the model can't fake

Engine version, prompt version, model, tier, filing accession and a UTC timestamp are appended in code after the model returns — so provenance can never be omitted, reworded or hallucinated.

Comparability flags

The traps that fool juniors

Every memo opens by reconciling its sources: GAAP vs non-GAAP, filing figures vs call figures, and an annual scorecard whose period may pre-date a quarterly filing. Mismatches are stated, not buried.

Safety

Sanitised, escaped output

The model-written body is HTML-sanitised before it is opened in headless Chrome for PDF export, so a <script> echoed out of a filing can't execute. Every dynamic value is escaped.

Honest tone

No green-by-default

Dashboard colour comes from the model's actual verdict or the real number — never from the word "verified." Negative free cash flow shows red; an un-judged ratio stays neutral, not falsely green.

Ingredient-based caching — pay only for what changed

The pipeline runs on flex pricing (~50% cheaper, slower) and caches on ingredients, not time. A per-ticker manifest records the inputs that produced each artifact; on the next run, only the steps whose inputs actually changed are re-executed.

06 · As a product

Research that lives inside your agent

The pipeline is exposed as a Model Context Protocol server — the bet being that the future is not another web app, but capabilities that plug straight into the tools people already drive all day: Claude Code, Cursor, Codex.

analyze_ticker

Full research run

Runs the entire pipeline for one ticker and returns the synthesised memo. Ingredient caching means a repeat call is near-free.

compare_tickers

Ranked screen

Analyses several names and returns a comparative ranking table with stance, conviction and a one-line rationale per company.

list / retrieve

Your research history

Lists prior analyses with dates, filing types and cost, and serves any cached report back instantly.

An agent calling these tools gets the same auditable artifacts a human would — the markdown memo for reading, the JSON brief as a UI contract — without the project ever needing to build or host a front end.
